GLM Coding Plan 抢购助手 + 本地自动验证码

这是一个用于智谱 GLM Coding Plan 页面流程辅助的油猴脚本，配合本地 CPU/GPU OCR 后端使用，可以自动识别中文点选验证码并完成点击。 项目同时提供 Windows 一键启动包和 macOS Apple Silicon 启动脚本。

主要功能：

  • 提前让套餐按钮可点击，减少手动刷新等待
  • 辅助 GLM Coding Plan 限流后的重试流程
  • 调用本地 OCR 后端识别中文点选验证码
  • 自动点击验证码图片中的目标文字
  • 支持 Rush mode 定时确认，默认目标时间为 10:00:00
  • 默认不自动点击订阅，避免脚本加载后主动进入购买链路；到点或用户按快捷键开启后才开始
  • 支持快捷键暂停/恢复脚本、切换自动点击订阅，并可在配置面板自定义快捷键
  • 支持配置面板、一键多开窗口、清除今日套餐状态缓存
  • 支持 Windows 一键启动后端；macOS Apple Silicon 可使用 .command 启动脚本
  • 默认直接打开 GLM Coding 页面（原内置折扣入口对应的邀请码活动已下线，代码已注释保留，活动恢复时取消注释即可）

抢购步骤：

1、先安装好油猴插件，配置好油猴脚本。使用 Chrome 插件时，要在页面右上角开启开发者模式，然后找到篡改猴插件点击详情，把允许运行用户脚本、在无痕模式下启用、允许访问文件网址都打开。
2、安装后端。Windows 用户可下载 GitHub Release 里的在线安装包或便携 CPU 包；macOS 用户请使用 Apple Silicon 机器，并按 docs/macos-setup.md 安装 Python 3.12 后运行 one-click-start.command。
3、打开抢购网址测试，看后端识别、验证码点击和页面辅助功能是不是正常。
4、每天 9 点 50 分前进入抢购页面准备，晚了可能就打不开。顺便准备好手机支付宝付款；我曾经遇到过有金额但付款慢了就没了。
5、默认不要让脚本过早自动点击订阅；到点后由 Rush mode 进入购买链路，或者你按快捷键显式开启自动点击。验证码可以先识别并点击好，但不要过早确认；Rush mode 会按目标时间保守释放确认。近期风控很严，建议只开一个窗口，开得越多越容易触发风控；根据实际情况在配置面板里把验证码点字延时调大（默认 250-400ms，可以试 300-450ms 甚至更大），节奏太快现在很容易被封。
6、如果这波没抢到，就盯着一个窗口用 OCR 继续识别点击。默认不会自动关闭支付页面。注意：如果看到没有金额的支付页面，那就是没抢到，要关掉继续抢。这时候可以使用快捷键进行快速操作。

快捷键：

  • Esc：关闭系统繁忙弹窗或支付弹窗
  • Enter / Space：点击验证码确认按钮
  • F8：暂停/恢复脚本；暂停后停止扫描、订阅点击、验证码自动点击/确认
  • Shift+F8：批量暂停/恢复（同浏览器所有 glm-coding 普通窗口联动；只切当前窗口用 F8）
  • F9：切换自动点击订阅

说明：F8/F9 是默认快捷键，可在配置面板中自定义。脚本会避开输入框、文本框、下拉框和可编辑区域，避免正常输入时误触发。Shift+F8 覆盖同浏览器普通窗口；无痕模式、跨浏览器受扩展隔离限制不支持。

重要提醒：

  • ⚠️ 2026-06-23 风控升级：智谱升级了风控，自动点击订阅按钮偶尔会被拦截/拒绝（连人手动点也一样会被拦），页面可能显示「很抱歉，由于您访问的URL有可能对网站造成安全威胁，您的访问被阻断」。应对：① 点击被拦时等约 10 秒重试；② 自动点击一直无效时手动点击「特惠订阅」入口；③ 验证码不用管——不管订阅入口怎么进的，验证码识别、点字、点确定都会自动接管。一句话：订阅入口点不动就手动点特惠订阅，验证码自动打。
  • 🔧 2026-06-26 v23.7 更新：修复 rush（冲刺）模式到目标时间后不自动点验证码「确定」的问题。如果你之前开了冲刺模式，到点验证码识别完却不自动确定，升级到 v23.7 即可解决（根因是验证码熔断在冲刺窗口前误触发且不可恢复，现已修复）。
  • 🔧 2026-07-09 v23.11 更新：① 新增 Shift+F8 批量暂停/恢复，按一下让同浏览器所有 glm-coding 窗口一起暂停或恢复，原 F8 单键行为不变；② 邀请码活动已下线，注释邀请码注入逻辑，打开页面不再注入 ?ic=... 参数。
  • 关于验证码「确定」按钮：① 冲刺模式（rush）到目标时间后会自动点确定；② 普通模式下默认不自动点确定，需要在配置面板「自动点击验证码确定」里手动开启。
  • 默认不会自动点击订阅按钮，避免脚本加载后主动触发购买链路；需要在配置面板或用 F9 开启。
  • 默认不会自动关闭无效支付链接或限流弹窗，需要在配置面板里手动开启。
  • 遇到真正有金额的支付二维码，请自行确认后再扫码支付。
  • macOS 版目前仅支持 Apple Silicon + CPU 后端，不支持 Windows 那套自动截图验证码弹窗；主流程通过油猴脚本直接发送验证码原图到本地后端。
  • 📢 官方近期不放量：近期智谱几乎不放量，连配套的「拼好模」活动也取消了，整体很难抢到。这不是脚本的问题，是官方供给侧收紧。脚本该做的优化（单窗口单发、实时验证码、请求节奏控制）都在做，但库存没了再快也没用。
  • 抢购是否成功受库存、限流、账号状态、支付速度等因素影响，脚本不能保证一定抢到。
